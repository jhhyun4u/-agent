# 제안 프로젝트 생성 디버깅 가이드

## 문제
"제안결정" 버튼을 클릭해도 제안 프로젝트가 생성되지 않음.

## 개선 사항
프론트엔드 에러 처리를 개선하여 이제 다음이 표시됩니다:
1. **화면에 에러 메시지 표시** - API가 실패하면 빨간 박스로 에러 메시지 표시
2. **상세한 콘솔 로그** - 개발자 도구 콘솔에서 각 단계의 상세 정보 기록

## 테스트 절차

### 1. 브라우저 DevTools 열기
```
F12 또는 우클릭 → 검사 → Console 탭
```

### 2. "제안결정" 버튼 클릭

### 3. 다음 중 하나가 표시될 것:

#### ✅ 성공 (원하는 결과)
```
[review] Token 상태: 빈 토큰 (DEV_MODE)
[review] 제안 프로젝트 생성 요청: http://localhost:8000/api/proposals/from-bid {bid_no: "..."}
[review] 제안 프로젝트 생성 응답 상태: 201
[review] 제안 프로젝트 생성 성공: [proposal_id]
```
→ /proposals 페이지로 자동 이동

#### ❌ 실패 (에러 메시지 포함)
```
[review] Token 상태: 빈 토큰 (DEV_MODE)
[review] 제안 프로젝트 생성 요청: http://localhost:8000/api/proposals/from-bid {bid_no: "..."}
[review] 제안 프로젝트 생성 응답 상태: [STATUS_CODE]
[review] 제안 프로젝트 생성 실패: [STATUS_CODE] {...error_details...}
```

화면에도 빨간색 에러 박스가 표시됨:
```
제안 프로젝트 생성 실패: [에러 메시지]
```

## 보고할 정보
콘솔 로그와 화면의 에러 메시지를 함께 보고하면 정확한 원인을 파악할 수 있습니다.

### 일반적인 에러 원인

| 에러 | 원인 | 해결책 |
|------|------|-------|
| 401 Unauthorized | 인증 실패 | 로그인 다시 시도 |
| 400 Bad Request | 요청 형식 잘못됨 | 콘솔 에러 메시지 확인 |
| 404 Not Found | 공고 정보 없음 | 공고 번호 확인 |
| 500 Internal Error | 서버 에러 | 백엔드 로그 확인 필요 |
| 네트워크 에러 | API 서버 연결 불가 | 백엔드 서버 실행 확인 |
