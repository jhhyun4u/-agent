# 제안 작업 워크플로우 UI 설계 사양

## 1. 제안 프로젝트 목록 (테이블 형식)

### 컬럼 구성

| 컬럼   | 설명                 | 타입                           |
| ------ | -------------------- | ------------------------------ |
| 과제명 | 제안 프로젝트 이름   | 링크 (클릭 시 워크플로우 전환) |
| 담당팀 | 담당 팀 이름         | 텍스트                         |
| PM     | Product Manager 이름 | 텍스트                         |
| PL     | Project Leader 이름  | 텍스트                         |
| 발주처 | 클라이언트 조직명    | 텍스트                         |
| 금액   | 예정가격 (₩)         | 숫자 (우측 정렬)               |
| 제출일 | 마감 기한            | 날짜 (D-Day 강조)              |
| 상태   | 진행 상태            | 배지 (색상 코딩)               |

### 상태 종류 및 색상

- **대기** (Waiting) — 회색 `#555555`
- **진행** (Processing) — 파란색 `#3b82f6`
- **중단** (Paused) — 주황색 `#f59e0b`
- **완료** (Completed) — 초록색 `#10b981`
- **제출** (Submitted) — 보라색 `#a855f7`
- **결과** (Results) — 분홍색 `#ec4899`
- **종료** (Closed) — 어두운회색 `#555555`

### 상호작용

- **행 클릭** → 해당 과제의 워크플로우 화면으로 전환
- **정렬** → 컬럼 헤더 클릭으로 정렬
- **필터** → 상태별 필터링 가능

## 2. 제안 작업 진행 (워크플로우)

### 레이아웃 구조

```
┌─────────────────────────────────────────────────────────────┐
│ Header: 과제명 | 발주처 | PM/PL | 상태 | 마감일          │
├──────────────┬──────────────────────────┬──────────────────┤
│              │                          │                  │
│  Step Nodes  │   Work Content (HITL)    │  AI Suggestions  │
│  (좌측)      │   (중앙)                 │  (우측)          │
│              │                          │                  │
│ • RFP 분석   │ [현재 단계 작업]         │ 📋 AI 제안      │
│ • Go/No-Go   │ [입력 폼]                │ 💡 가이드       │
│ • 전략 수립  │ [콘텐츠 편집]           │ ✓ 체크리스트    │
│ • 계획 수립  │ [HITL 버튼]             │ ⚠️ 알림         │
│ • 제안서     │                          │                  │
│ • PPT        │ [승인] [반려] [수정요청] │                  │
│ • 평가 모의  │                          │                  │
│ • 최종 제출  │                          │                  │
│              │                          │                  │
└──────────────┴──────────────────────────┴──────────────────┘
```

### 좌측: Step Nodes (단계 노드)

**구성**

- 8개 메인 단계 표시
- 현재 단계: 강조 표시 (배경색 변경, 텍스트 밝음)
- 완료 단계: 체크마크 ✓ 표시
- 대기 단계: 회색 처리

**단계별 설명**

```
0️⃣ RFP 분석 (Step: rfp_analyze)
   └─ G2B 공고 분석, Compliance Matrix 생성

1️⃣ Go/No-Go (Step: go_no_go)
   └─ 3축 정량 평가 (유사실적+자격+경쟁)

2️⃣ 전략 수립 (Step: strategy_generate)
   └─ 포지셔닝 매트릭스, Win Theme 도출

3️⃣ 계획 수립 (Step: plan_story)
   └─ 목차, 스토리라인, 팀 구성 결정

4️⃣ 제안서 작성 (Step: proposal_write)
   └─ 섹션별 순차 작성, AI 어시스턴트

5️⃣ PPT 작성 (Step: ppt_slide)
   └─ 발표 자료 생성

6️⃣ 평가 모의 (Step: evaluation_simulation)
   └─ 발주처 평가 시뮬레이션

7️⃣ 최종 제출 (Step: final_submit)
   └─ 품질 검증 후 제출
```

## 4. 기술 구현 방안

### 데이터 흐름

```
테이블 행 클릭
  ↓
proposalId 취득
  ↓
/api/proposals/{id}/workflow 호출
  ↓
ProposalState 로드 (단계, 진행상황, AI 결과)
  ↓
워크플로우 화면 렌더링
```

### HITL 상호작용

```
[승인] 클릭
  ↓
POST /api/workflow/{id}/approve
  ↓
다음 단계로 이동
  ↓
Step Node 업데이트
```

### Step Node 상태 관리

```
const steps = [
  { id: 0, label: 'RFP 분석', status: 'completed' },
  { id: 1, label: 'Go/No-Go', status: 'active' },
  { id: 2, label: '전략 수립', status: 'pending' },
  ...
]

상태: 'completed' | 'active' | 'pending' | 'error'
```

## 5. UI 컴포넌트

### Reusable Components

- `StepNode` — 단계 노드 (상태 표시)
- `StatusBadge` — 상태 배지 (색상 코딩)
- `WorkContentPanel` — 작업 콘텐츠 영역
- `AIPanel` — AI 제안 패널
- `HITLButtons` — 승인/반려/수정 버튼

## 6. 반응형 디자인
