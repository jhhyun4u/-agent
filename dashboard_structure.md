## 📋 위젯 목록 (위젯 설정 메뉴)

위젯 설정 버튼 (⚙️)에서 on/off 가능:

| # | 키 | 레이블 | 기능 | 기본값 |
|---|---|--------|------|-------|
| 1 | `action` | 지금 해야 할 것 | 긴급 일정 + 진행 중 제안서 | ON |
| 2 | `pipeline` | 제안 파이프라인 | 6단계 파이프라인 상태 | ON |
| 3 | `kpi` | 제안 분석 KPI | 올해/이번달/지난달 KPI 카드 | ON |
| 4 | `charts` | 분석 차트 | 월별 추이 + 낙찰실패 원인 | ON |
| 5 | `client` | 기관별 수주 | 발주기관별 수주율 막대 차트 | ON |
| 6 | `team` | 팀별 성과 | 팀별 성과 + 포지셔닝별 수주율 | ON |
| 7 | `positioning` | 포지셔닝별 | (팀별 성과와 통합됨) | ON |
| 8 | `bids` | 공고 모니터링 | 추천 공고 (S/A 등급) | ON |
| 9 | `calendar` | RFP 캘린더 | 월간 캘린더 + 일정 추가 | ON |

## 🎯 스코프별 표시 (상단 탭)

### scope = "personal" (개인)
- 본인의 진행 중인 제안서만 표시
- 결재 대기, 마감 임박 표시 안 됨

### scope = "team" (팀)
- **결재 대기** 섹션 표시 (팀장용 결재 진행 상황)
- **마감 임박** 섹션 표시 (팀 내 긴급 공고)
- 팀원들의 제안서/일정 통합 보기

### scope = "division" (본부)
- 본부 내 모든 팀의 데이터
- 팀별 성과 비교 표시

### scope = "company" (전체)
- **본부별 성과 비교** 섹션 표시 (경영진용)
- 전사 통합 데이터

## ❓ 질문: 필터와 "+RFP업로드" 필요?

### "+RFP업로드" 버튼 ❌ 불필요
- 대시보드는 **보기 전용** 화면
- RFP 등록은 별도 메뉴(/monitoring, /search)에서 처리
- 대시보드에서 빠른 업로드는 UX 혼란 야기

### 추가 필터 (팀/본부/전체에서만)
- ✅ 현재 구현: **스코프 탭** (개인/팀/본부/전체)
- 💬 검토 필요:
  - **팀/본부/전체** 스코프에서 추가 필터 필요?
    - 상태별 필터? (진행중, 완료, 수주, 낙찰실패)
    - 기간 필터? (월별, 분기별)
    - 담당자 필터?
    - 기관 필터?

## 설계 의도 정리

1. **개인**: 본인의 TODO 리스트 (할 일 + 진행 중)
2. **팀장 (팀 스코프)**: 팀원 관리 (결재 대기, 마감 임박, 성과)
3. **경영진 (전체 스코프)**: 조직 전체 KPI + 본부별 비교

**현재 상태**: 각 스코프별로 역할에 맞는 정보만 표시 ✅
