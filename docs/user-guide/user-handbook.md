## 기본 개념

### 제안서 (Proposal)
**정의**: 공고에 응하기 위해 작성하는 공식 제출 문서

**상태 흐름**:
```
초안 (Draft)
  ↓
검토 중 (Under Review)
  ↓
승인됨 (Approved)
  ↓
제출됨 (Submitted)
  ↓
낙찰 / 낙선 (Won / Lost)
```

### 워크플로 (Workflow)
**정의**: 제안서 작성 과정을 단계적으로 진행하는 자동화된 프로세스

**주요 단계**:
- **Step 0**: RFP 분석 (공고 수집 및 분석)
- **Step 1**: 전략 수립 (Go/No-Go 판정, 포지셔닝)
- **Step 2**: 계획 (팀 구성, 일정, 비용)
- **Step 3**: 제안서 작성 (본문 및 부록)
- **Step 4**: 검토 및 승인 (자가 진단, 최종 검토)
- **Step 5**: 제출 (최종 확인 및 제출)

### AI Coworker (에이전트)
**역할**: 
- 자동 작성 (문안 생성, 기초 분석)
- 의견 제시 (전략 평가, 위험 식별)
- 지식 조회 (과거 제안, 경쟁사 정보)

**상호작용 방식**:
- **AI 완료 알림**: "이 섹션이 AI에 의해 완성되었습니다. 검토 및 수정해주세요."
- **AI 제안**: "이 부분을 다시 쓰시겠습니까?" 버튼
- **AI 인터럽트**: 필요시 AI 실행 일시 중지 및 수동 편집 가능

## 프로젝트 생성 및 관리

### 새 프로젝트 생성

#### 방법 1: 공고 검색 (G2B)
1. 홈 → "새 제안서" → "공고 검색"
2. 키워드 검색 (예: "정보시스템", "구축", "2026년")
3. 공고 선택
4. AI가 자동으로 "적합도 점수" 제시
5. "프로젝트 생성" 클릭
6. 프로젝트 명 확인/수정
7. "완료" 클릭

#### 방법 2: 수동 입력
1. 홈 → "새 제안서" → "수동 입력"
2. 공고 정보 입력:
   - 발주처 이름
   - 사업 명
   - 마감일
   - 예상 가격
3. "만들기" 클릭

### 프로젝트 설정

프로젝트 상세 페이지에서:

**기본 정보**
- 발주처: 공고 데이터에서 자동 입력
- 사업 명: 수동 수정 가능
- 예상 가격: 초기 가격 설정
- 마감일: 제출 기한 설정

**팀 구성**
- 프로젝트 리더 선택
- 팀원 추가 (부서, 역할 지정)
- 권한 설정 (편집, 읽기, 검수)

**AI 설정**
- AI 자동 실행 여부
- AI 깊이 수준 (Basic / Advanced / Expert)
- 선호 작성 스타일

## AI 협업 흐름

### 단계 1: RFP 분석

**화면**: "RFP 분석" 탭

```
┌─────────────────────────────────────────┐
│ RFP 분석 결과                            │
├─────────────────────────────────────────┤
│ 발주처: OO 부처                          │
│ 예산: 30억원                            │
│ 마감: 2026-06-30                        │
│                                         │
│ [Compliance Matrix 보기]                │
│ [위험 요소 분석]                         │
│ [다음 단계로 진행]                      │
└─────────────────────────────────────────┘
```

**주요 기능**:
- **Compliance Matrix**: 공고 요구사항 vs 우리 강점 매핑
- **위험 요소 분석**: 제약사항, 기술적 위험 식별
- **비용 추정**: AI가 자동 계산한 예상 비용

**조작 방법**:
1. 공고 파일 업로드 (PDF, HWP, HWPX)
2. "분석 시작" 클릭 → AI가 2~3분 소요
3. 결과 검토
4. "Go/No-Go 판정"에서 진행 여부 결정
5. Go 선택 시 다음 단계 자동 활성화

---

### 단계 2: 전략 수립

**화면**: "전략" 탭

```
┌─────────────────────────────────────────┐
│ 포지셔닝 전략                            │
├─────────────────────────────────────────┤
│                                         │
│ 우리의 Win Theme:                      │
│ "효율적이고 신뢰할 수 있는              │
│  정보시스템 구축 파트너"                 │
│                                         │
│ [수정]  [AI 제안]  [검토 완료]          │
│                                         │
│ ───────────────────────────────────────┤
│ SWOT 분석                               │
│ [강점]  [약점]  [기회]  [위협]          │
│                                         │
│ ───────────────────────────────────────┤
│ 경쟁 분석                               │
│ 예상 경쟁사: ○○사, □□사                │
│ 우리의 차별점: △△△                     │
│                                         │
└─────────────────────────────────────────┘
```

**주요 활동**:
1. **포지셔닝**: 공고와의 적합성 정의
2. **SWOT 분석**: AI가 초안 제시, 팀이 검증
3. **경쟁 분석**: AI가 자동 조회 (지난 낙찰 정보)
4. **가격 전략**: 비용 시뮬레이션 (±10% 범위)

**팁**:
- "AI 제안" 버튼으로 AI의 대안 확인
- 과거 유사 제안과 비교 (우측 "유사 사례" 패널)
- 주의: 과도한 가격 인하는 피할 것

### 단계 3: 계획 수립

**화면**: "계획" 탭 (4개 섹션 병렬)

```
[팀 구성]  [일정]  [스토리라인]  [가격]
```

#### 팀 구성
- 역할별 인원 배치 (PM, 기술팀장, 개발자 등)
- AI 권장 구성 확인
- 최종 팀원 승인

#### 일정
- 마일스톤 설정 (기획, 설계, 구현, 테스트)
- AI가 자동 계산한 일정 검토
- 마감일 역계산

#### 스토리라인
- 제안서의 "이야기" 뼈대
- 섹션별 논거 (예: 1장 제안 개요 → 2장 기술 전략 → ...)
- AI의 초안 활용 후 팀에서 수정

#### 가격
- 인건비, 장비, 외주 비용 입력
- AI가 시장 상황 반영한 조정안 제시
- 이익률 설정 (표준: 15~25%)

### 단계 4: 제안서 작성

**화면**: "작성" 탭 (10개 섹션)

```
1. 제안 개요
2. 발주처 이해 및 제안 방향
3. 기술 전략
4. 기술 설계
5. 구현 계획
6. 품질 보증
7. 위험 관리
8. 일정 및 인력
9. 가격 제안
10. 회사 소개
```

**각 섹션 작성 흐름**:

1. "AI 초안 생성" 클릭
   - AI가 2~5분 내에 초안 작성
   - Slack / 인앱 알림 수신

2. 초안 검토
   ```
   [AI 초안 내용]
   
   ✓ 좋음 / ✗ 수정 필요 / ? 더 보기
   ```

3. 수정 (온라인 에디터)
   - 문장 직접 수정
   - Markdown 지원
   - 실시간 자동 저장

4. 피드백 → AI 재작성
   ```
   "이 부분을 더 기술적으로 상세히 작성해주세요"
   → [AI 재작성 실행]
   ```

5. 완료 → 다음 섹션

**팁**:
- 각 섹션마다 "출처 확인" 기능으로 AI가 사용한 정보 추적
- "자가 진단" 기능으로 섹션 품질 자동 검사
- 버전 관리: 모든 편집 이력 저장 (필요시 이전 버전 복구)

- 작성자가 수정 후 재제출

**최종 승인자 (Approver)**:
- 검수 완료 후 최종 승인
- 승인 시 "제출 가능" 상태 변경

## 일반적인 작업

### 작업 1: 과거 제안 검색

**시나리오**: "유사한 발주처에 이전에 제출한 제안이 있나?"

**단계**:
1. 홈 → "지식베이스" → "제안서 아카이브"
2. 발주처명 또는 사업명으로 검색
3. 결과 선택 → 상세 보기
4. "이 제안을 템플릿으로 사용" 버튼
5. 새 프로젝트에서 기존 내용 자동 적용

### 작업 2: 경쟁사 정보 조회

**시나리오**: "경쟁 예상 업체의 최근 낙찰 정보를 알고 싶어"

**단계**:
1. 프로젝트 → "전략" 탭
2. "경쟁 분석" 섹션 → "경쟁사 낙찰 정보"
3. AI가 자동 조회 (G2B, 관공서 뉴스)
4. 최근 낙찰가, 가격 대비 등 분석
5. "우리 가격 추천안" 조회

### 작업 3: 문서 다운로드 및 편집

**시나리오**: "작성된 제안서를 Word로 다운로드해서 추가 편집하고 싶어"

**단계**:
1. 프로젝트 → "제출" 탭
2. "다운로드" 버튼
3. 형식 선택: DOCX / PDF / HWP
4. 다운로드 완료 후 로컬에서 편집
5. 편집 완료 후 다시 업로드 (수동)

### 작업 4: 팀원과 실시간 협업

**시나리오**: "여러 팀원이 동시에 같은 섹션을 편집하고 싶어"

**방법**:
- 플랫폼의 온라인 에디터 사용 (자동 동기화)
- 충돌 발생 시: 최신 버전 기준으로 자동 병합
- 변경 이력: "히스토리" 탭에서 모든 편집 추적

### 작업 5: AI 작업 일시 중지 및 수동 편집

**시나리오**: "AI가 생성한 내용이 마음에 안 들어서 직접 처음부터 작성하고 싶어"

**단계**:
1. 섹션 → "AI 초안" 우측 "⏹ 일시 중지" 버튼 클릭
2. AI 작업 중단 (완료 후에만 가능)
3. 온라인 에디터에서 직접 작성
4. "저장" 클릭
5. 나중에 다시 AI 활용 가능

## FAQ

### Q1: AI가 생성한 내용이 정말 정확한가요?
**A**: AI는 과거 제안서, 뉴스, 공식 문서를 기반으로 초안을 생성합니다. **항상 검수 필수**입니다.
- 사실 확인 (특히 경쟁사 정보, 기술 사양)
- 회사 브랜드 톤 맞춤
- 발주처 특성 반영
**권장**: 최소 2명 이상이 검수

### Q2: 제안 마감이 얼마 남지 않았는데 빠르게 진행할 수 있나요?
**A**: 네, 다음 방법으로 시간 단축 가능합니다:
1. AI 깊이 설정 → "Express" 모드 (표준 → 30% 빠름)
2. 팀 병렬 작업: 각 섹션을 다른 팀원이 동시 편집
3. 템플릿 사용: "유사 제안 템플릿" 적용 후 커스터마이징
4. AI 피드백 활용: 작성 → 1시간 후 AI 재검토 → 수정

### Q3: 개인적으로 생성한 내용이 회사에 자산화되나요?
**A**: 네, 다음과 같이 자동 관리됩니다:
- 모든 제안서는 자동 저장 (용역제안DB)
- 낙찰 시: 자동으로 "지식베이스" → "성공 사례" 이동
- 낙선 시: "실패 사례" 분류 (교훈 추출용)
- 개인 정보 보호: 작성자 익명화 가능

### Q4: 제안 중단 후 다시 시작할 수 있나요?
**A**: 네, 언제든 가능합니다:
1. 홈 → "진행 중인 프로젝트"
2. 중단된 프로젝트 선택
3. "재개" 버튼 → 마지막 저장 지점부터 시작
4. 모든 이전 내용 그대로 보존

### Q5: 보안은 어떻게 보장되나요?
**A**: 다음과 같이 관리됩니다:
- **전송**: HTTPS 암호화
- **저장**: AES-256 암호화
- **접근**: 역할 기반 (RBAC) 제어
- **감사**: 모든 접근/편집 로그 기록
- **백업**: 일일 자동 백업 (7일 보관)

## 문제 해결

### 문제 1: 로그인이 안 돼요

**증상**: "로그인 실패" 메시지

**해결 순서**:
1. 브라우저 캐시 삭제 (Ctrl+Shift+Delete)
2. 다른 브라우저 시도 (Chrome, Firefox, Edge)
3. "비밀번호 재설정" 클릭
4. 재설정 링크 메일 수신 → 새 암호 설정
5. **여전히 안 되면**: ops-support@tenopa.io 연락

**팁**: Microsoft 계정이 여러 개인 경우, 조직 계정(@tenopa.io) 확인

### 문제 2: AI가 예상 시간보다 오래 걸려요

**증상**: "처리 중..." 30분 이상 표시

**원인 및 해결**:
| 원인 | 해결 |
|------|------|
| 과도한 로드 | 잠시 후 다시 시도 |
| 네트워크 불안정 | 연결 확인 후 새로고침 (F5) |
| 너무 긴 문서 | 공고를 이미지가 아닌 텍스트로 변환 |
| 브라우저 문제 | 다른 브라우저 사용 |

**강제 종료**: 30분 이후 자동으로 작업 중단 → 수동 재시도

### 문제 3: 저장되지 않은 내용이 사라졌어요

**증상**: "작성한 내용이 없어졌어요"

**복구 방법**:
1. 프로젝트 → "히스토리" 탭
2. 시간순 편집 목록 표시
3. 사라진 시점 **직전**의 버전 선택
4. "이 버전으로 복원" 클릭
5. 복원됨 → 다시 저장

**예방**: 중요한 내용 작성 후 **Ctrl+S** (수동 저장)

### 문제 4: 팀원이 동시에 편집할 수 없어요

**증상**: "이미 누군가 이 섹션을 편집 중입니다" 메시지

**원인**: 온라인 에디터의 동시 편집 잠금

**해결**:
1. 다른 팀원의 편집 완료 대기 (자동으로 5분 후 해제)
2. 또는: "강제 해제" 클릭 (주의: 상대방 내용 손실 가능)
3. 대신: 다른 섹션 작업 후 나중에 시도

**팁**: 대규모 팀 작업 시 → "편집 일정" 공유 (각 섹션별 담당자)

### 문제 5: PDF 다운로드가 안 돼요

**증상**: "다운로드 실패" 또는 손상된 파일

**해결**:
1. 브라우저의 다운로드 폴더 확인
2. 다시 다운로드 시도 (다른 브라우저)
3. "PDF 미리보기"로 먼저 확인
4. **여전히 안 되면**: ops-support@tenopa.io 연락 (관리자가 서버에서 생성)
