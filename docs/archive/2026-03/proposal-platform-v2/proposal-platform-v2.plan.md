---

## 1. 배경 및 목적

v1은 단일 RFP → 제안서 생성 도구였다면, **v2는 팀 단위 제안서 관리 플랫폼**으로 확장.
반복 작업을 줄이고, 팀 지식을 축적하며, 수주율을 높이는 것이 목표.

---

## 2. 구현 범위 (순차 구현)

### Phase A — 섹션 라이브러리 + 아카이브 (임팩트 최대)
회사 자산을 AI에게 먹이면 제안서 품질이 즉시 향상됨.

| 기능 | 설명 |
|------|------|
| 섹션 라이브러리 | 회사소개·수행실적·기술방법론 등 재사용 블록 관리 |
| 회사 자료 업로드 | PDF/DOCX 업로드 → AI 분석 → 섹션 자동 추출 |
| 제안서 생성 연동 | 새 제안서 생성 시 섹션 선택 → AI에 컨텍스트 주입 |
| 아카이브 뷰어 | 회사 전체/팀/개인 제안서 이력 + 수주결과 필터 |

### Phase B — 작업 단계 + 버전관리 (UX 핵심)
진행 중인 제안서의 상태를 실시간으로 파악하고, 단계별 결과물을 관리.

| 기능 | 설명 |
|------|------|
| 단계 대시보드 | Phase 1~5 진행 상태 + 예상 완료 시각 |
| 단계별 결과물 뷰어 | 각 Phase 완료 시 JSON/텍스트 결과물 확인 |
| 버전 관리 | 동일 RFP로 재생성 시 v1/v2/v3 관리 |
| 단계 재시작 | 특정 Phase부터 재실행 |

### Phase C — 공통서식 라이브러리
부처별·기관별 요구 서식을 모아두고 제안서 생성 시 적용.

| 기능 | 설명 |
|------|------|
| 서식 업로드 | DOCX/PDF 서식 파일 업로드 + 메타정보 태깅 |
| 부처/기관 분류 | 행안부·과기부·교육부 등 카테고리 |
| 서식 선택 | 새 제안서 생성 시 서식 선택 → AI 적용 |
| 공개/비공개 | 팀 내 공유 or 전체 공개 |

### Phase D — 수주율 대시보드 + RFP 캘린더
| 기능 | 설명 |
|------|------|
| 수주율 통계 | 기관별·유형별·팀원별 수주율 차트 |
| RFP 캘린더 | 마감일 등록 + D-n 알림 |
| 유사 RFP 추천 | 과거 수주 제안서 유사도 매칭 |

## 3. 성공 기준

| Phase | 기준 |
|-------|------|
| A | 섹션 라이브러리에서 선택한 내용이 AI 생성 제안서에 반영됨 |
| A | 아카이브에서 회사/팀/개인 필터가 동작함 |
| B | Phase 진행 대시보드가 실시간으로 업데이트됨 |
| B | 동일 RFP의 v1/v2 결과물을 나란히 비교 가능 |
| C | 서식 선택 후 생성된 제안서에 서식이 반영됨 |
| D | 수주율 차트가 실제 win_result 데이터 기반으로 표시됨 |

## 4. 기술 영향 범위

### DB (Supabase)
- `sections` 테이블 신규 (섹션 라이브러리)
- `company_assets` 테이블 신규 (회사 자료)
- `form_templates` 테이블 신규 (공통서식)
- `proposals` 테이블: `version`, `parent_id`, `form_template_id` 컬럼 추가
- `rfp_calendar` 테이블 신규

### 백엔드 (FastAPI)
- `/api/sections` CRUD
- `/api/assets` 업로드/조회
- `/api/templates` 서식 관리
- `phase_executor`: 섹션 컨텍스트 주입 로직 추가
- `/api/proposals/{id}/versions` 버전 목록

### 프론트엔드 (Next.js)
- `/resources` — 섹션 라이브러리 + 자료 업로드
- `/resources/templates` — 서식 라이브러리
- `/archive` — 아카이브 뷰어
- `/proposals/[id]` — 단계 대시보드 개선 + 버전 탭
- `/proposals/new` — 섹션/서식 선택 단계 추가
- `/dashboard` — 수주율 차트 + 캘린더

## 6. 구현 순서

```
Phase A (섹션 + 아카이브)
  → Phase B (단계 대시보드 + 버전관리)
    → Phase C (서식 라이브러리)
      → Phase D (수주율 + 캘린더)
```

각 Phase 완료 후 Gap Analysis → 90% 이상 시 다음 Phase 진행.
