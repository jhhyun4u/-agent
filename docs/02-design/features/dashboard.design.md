# 대시보드 설계 문서 (Dashboard)

**버전**: v1.0
**작성일**: 2026-03-29
**상태**: APPROVED

---

## 1. 개요

대시보드는 팀장/본부장/경영진을 위한 **통합 모니터링 및 성과 분석 화면**입니다.
개인 단위 작업 추적은 제안서 상세 페이지와 제안 목록에서 관리하며, 대시보드는 팀/본부/전사 관점의 KPI와 실시간 알림에 집중합니다.

---

## 2. 설계 원칙

| 원칙 | 설명 |
|------|------|
| **역할 기반** | 팀장(팀 데이터), 본부장(본부 데이터), 경영진(전사 데이터) |
| **한 화면 4개 섹션** | 긴급 알림 → 성과 요약 → 분석 → 세부 정보 |
| **클릭 이동** | 각 위젯의 데이터 → 상세 페이지로 드릴다운 |
| **설정 가능** | 위젯 on/off로 개인화 (9개 위젯) |
| **보기 전용** | 데이터 입력/수정은 다른 페이지에서 |

---

## 3. 레이아웃 구조

### 3.1 헤더 (Header)

```
┌─────────────────────────────────────────────────┐
│ 대시보드   ⚙️ 위젯설정  [팀] [본부] [전체]         │
└─────────────────────────────────────────────────┘
```

**좌측**: 페이지 제목 ("대시보드")
**우측**:
- **위젯 설정** (⚙️ 아이콘) — 드롭다운: 9개 위젯 체크박스
- **스코프 탭** (3개) — "팀" / "본부" / "전체" 중 1개 선택
  - 기본값: 사용자 권한에 따라 최상위 스코프로 설정
  - 팀장: 기본 "팀"
  - 본부장: 기본 "본부"
  - 경영진: 기본 "전체"

### 3.2 메인 컨텐츠 (Main)

**섹션 순서:**

1. **긴급 알림 (조건부)**
   - 결재 대기 (scope=팀)
   - 마감 임박 경고 (scope=팀)
   - 본부별 성과 비교 (scope=전체)

2. **오늘 할 일** (widgetConfig.action)
   - 긴급 공고 (D-14 이내)
   - 진행 중 제안서

3. **제안 파이프라인** (widgetConfig.pipeline)
   - 6단계: 공고등록 → 작성중 → 완료 → 결과대기 → 수주 → 낙찰실패

4. **제안 분석 KPI** (widgetConfig.kpi)
   - 3개 카드: 올해(YTD) / 이번달(MTD) / 지난달
   - 각 카드: 진행건 / 수주건 / 수주율

5. **분석 차트** (widgetConfig.charts)
   - 월별 수주율 추이 (선 그래프)
   - 낙찰 실패 원인 분석 (파이 차트)

6. **기관별 수주 현황** (widgetConfig.client)
   - 발주기관별 수주율 (수평 막대 차트)

7. **팀별 성과 + 포지셔닝별 수주율** (widgetConfig.team)
   - 팀별 성과 (카드)
   - 포지셔닝별 수주율 (카드)

8. **공고 모니터링** (widgetConfig.bids)
   - S/A 등급 추천 공고
   - 현재 제안서와 매칭되는 공고

9. **RFP 캘린더** (widgetConfig.calendar)
   - 월간 캘린더 + D-day 표시
   - 마감 없는 공고 일정 추가 기능

## 4. 스코프별 데이터 범위

### 4.1 scope = "팀" (팀장용)

**가시성:**
- 본인의 팀 데이터만
- 팀원들의 제안서, 일정 통합

**표시 섹션:**
- ✅ 결재 대기 (팀 내 대기 중 항목)
- ✅ 마감 임박 (팀 내 D-7 이내)
- ✅ 오늘 할 일 (팀 내 할 일)
- ✅ 파이프라인 (팀 전체)
- ✅ KPI (팀 전체)
- ✅ 분석 차트 (팀 전체)
- ✅ 기관별 수주 (팀 전체)
- ✅ 팀별 성과 (자신의 팀만)
- ✅ 공고 모니터링 (팀 할당 공고)
- ✅ RFP 캘린더 (팀 전체)

**권한:**
- 조회만 가능
- 팀원 제안서 상세는 별도 링크

### 4.2 scope = "본부" (본부장용)

**가시성:**
- 본부 내 모든 팀 데이터
- 팀 간 성과 비교

**표시 섹션:**
- ❌ 결재 대기 (팀장용)
- ❌ 마감 임박 경고 (팀장용)
- ✅ 본부별 성과 비교 (→ 팀별 성과 비교)
- ✅ 오늘 할 일 (본부 내 전체)
- ✅ 파이프라인 (본부 전체)
- ✅ KPI (본부 전체)
- ✅ 분석 차트 (본부 전체)
- ✅ 기관별 수주 (본부 전체)
- ✅ 팀별 성과 (본부 내 모든 팀)
- ✅ 공고 모니터링
- ✅ RFP 캘린더

**권한:**
- 조회만 가능

### 4.3 scope = "전체" (경영진용)

**가시성:**
- 전사 통합 데이터
- 본부 간 성과 비교

**표시 섹션:**
- ❌ 결재 대기 (팀장용)
- ❌ 마감 임박 경고 (팀장용)
- ✅ 본부별 성과 비교 (전사 + 본부별 비교 테이블)
- ✅ 오늘 할 일 (전사 긴급)
- ✅ 파이프라인 (전사 전체)
- ✅ KPI (전사 전체)
- ✅ 분석 차트 (전사 전체)
- ✅ 기관별 수주 (전사 전체)
- ✅ 팀별 성과 (모든 팀)
- ✅ 공고 모니터링 (추천 공고 통합)
- ✅ RFP 캘린더 (전사 공고)

**권한:**
- 조회만 가능

## 5. 위젯 명세

### 5.1 위젯 설정 메뉴

**인터페이스:**
- 우측 상단 ⚙️ 아이콘 클릭 → 드롭다운
- 9개 위젯 체크박스 (on/off)
- 선택사항 localStorage에 저장

**위젯 목록:**

| # | 키 (key) | 레이블 | 설명 | 기본값 |
|---|---------|--------|------|-------|
| 1 | action | 지금 해야 할 것 | 긴급 공고(D-14) + 진행 중 제안서 | ON |
| 2 | pipeline | 제안 파이프라인 | 6단계 파이프라인 상태 | ON |
| 3 | kpi | 제안 분석 KPI | 올해/이번달/지난달 카드 | ON |
| 4 | charts | 분석 차트 | 월별 추이 + 실패원인 | ON |
| 5 | client | 기관별 수주 | 발주기관별 수주율 막대 차트 | ON |
| 6 | team | 팀별 성과 | 팀별 성과 + 포지셔닝별 수주율 | ON |
| 7 | positioning | 포지셔닝별 | (팀별 성과에 통합) | ON |
| 8 | bids | 공고 모니터링 | S/A 등급 추천 공고 | ON |
| 9 | calendar | RFP 캘린더 | 월간 캘린더 + 일정 추가 | ON |

## 6. 각 위젯 상세 설계

### 6.1 결재 대기 (scope=팀만)

**언제 표시:**
- scope = "팀"
- 결재 대기 항목 > 0건

**내용:**
```
🟡 결재 대기   N건
├─ 제안서 제목 1          D-2
├─ 제안서 제목 2          D-5
└─ ...
```

**상호작용:**
- 클릭 → `/proposals/{id}` 상세 페이지
- D-day 색상: D-3 이내(빨강), D-14 이내(노랑), 그 외(회색)

---

### 6.2 마감 임박 경고 (scope=팀만)

**언제 표시:**
- scope = "팀"
- D-0 ~ D-7 공고 있을 때

**내용:**
```
! 마감 임박   N건 (D-7 이내)
├─ D-1  공고 제목 1
├─ D-3  공고 제목 2
└─ ...
```

**상호작용:**
- 클릭 → 관련 제안서 또는 공고 상세
- D-day 표시

---

### 6.3 본부별 성과 비교 (scope=전체만)

**언제 표시:**
- scope = "전체"
- 본부별 성과 데이터 있을 때

**내용:**
```
본부별 성과 비교
┌──────────────────────────────────────┐
│ 본부/팀    진행건  수주율  평균소요  전월대비 │
├──────────────────────────────────────┤
│ A팀       5건    60%     15일    +10%  │
│ B팀       3건    40%     20일    -5%   │
└──────────────────────────────────────┘
```

**정렬:** 수주율 내림차순
**상호작용:** 행 클릭 → 팀 상세

---

### 6.4 지금 해야 할 것 (widgetConfig.action)

**조건:** actionItems > 0건

**내용:**
- 긴급 공고 (D-0 ~ D-14)
- 진행 중 제안서 (status=processing)

**표시:**
```
🟢 지금 해야 할 것
├─ [D-1] 공고 제목       [지금 시작]
├─ [생성중] 제안 제목 1  [확인]
└─ [대기] 제안 제목 2    [시작]
```

**상호작용:**
- "지금 시작" → `/proposals/new` (새 제안 생성)
- "확인"/"시작" → `/proposals/{id}` 상세

---

### 6.5 제안 파이프라인 (widgetConfig.pipeline)

**레이아웃:** 수평 카드 6개

```
┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐
│공고  │ │작성  │ │완료  │ │결과  │ │수주  │ │낙찰  │
│등록  │ │중    │ │      │ │대기  │ │ ✓    │ │실패  │
│  5  │ │  3   │ │  8   │ │  2   │ │  4   │ │  1   │
└──────┘ └──────┘ └──────┘ └──────┘ └──────┘ └──────┘
   ↓       ↓       ↓       ↓       ↓       ↓
  /monitoring  /proposals (status=processing)  /archive
```

**색상 코드:**
- 공고등록: 회색 (#8c8c8c)
- 작성중: 파랑 (#3b82f6)
- 완료: 흰색 (#ededed)
- 결과대기: 노랑 (#fbbf24)
- 수주: 초록 (#3ecf8e)
- 낙찰실패: 빨강 (#f87171)

---

### 6.6 제안 분석 KPI (widgetConfig.kpi)

**레이아웃:** 3개 카드 (그리드)

```
┌─────────────────┐┌─────────────────┐┌─────────────────┐
│ 올해 (YTD)      ││ 이번달 (MTD)    ││ 지난달           │
│ 진행: 10건      ││ 진행: 3건       ││ 진행: 7건       │
│ 수주: 4건 (40%) ││ 수주: 1건(33%)  ││ 수주: 3건(43%)  │
└─────────────────┘└─────────────────┘└─────────────────┘
```

**데이터 계산:**
- YTD: 1월1일 ~ 오늘
- MTD: 1일 ~ 오늘 (이번 달)
- 지난달: 저번달 1일 ~ 말일

---

### 6.7 분석 차트 (widgetConfig.charts)

**레이아웃:** 2열 그리드

```
[월별 수주율 추이 선 그래프] [낙찰 실패 원인 파이 차트]
```

**월별 추이:**
- X축: 지난 12개월
- Y축: 수주율 (%)
- 색상: 초록 (#3ecf8e)

**낙찰 실패 원인:**
- 데이터: 최근 100건 제안서
- 항목: 가격경쟁력, 기술력, 입찰 누락, 기타 등
- 색상: 카테고리별 다색

```
발주기관       수주율
교육부   ████████████ 60%
국방부   ██████████ 50%
```

**정렬:** 수주율 내림차순
**상호작용:** 기관 클릭 → 기관별 제안 목록

### 6.10 공고 모니터링 (widgetConfig.bids)

**언제 표시:**
- S or A 등급 추천 공고 > 0건

**내용:**
```
공고 모니터링

[추천 공고 목록]
```

**상호작용:**

```
         1  2  3  4  5
```

- 마감 없는 공고 일정 추가
- 입력 후 RFP 마감일 업데이트

**상호작용:**
- 날짜 클릭 → 해당 공고 상세

## 7. 데이터 흐름

### 7.1 데이터 소스

| 위젯 | 데이터 테이블 | 필터 |
|------|-------------|------|
| 결재 대기 | proposals | status IN (paused, on_hold) |
| 마감 임박 | calendar_items | deadline BETWEEN now() AND now()+7days |
| 본부별 성과 | proposal_results + mv_team_performance | org_id, 팀별 그룹핑 |
| 파이프라인 | proposals | org_id, status 카운트 |
| KPI | proposals | org_id, created_at 범위 |
| 분석 차트 | proposals + proposal_results | org_id, 월별 그룹핑 |
| 기관별 수주 | proposals | org_id, client_name |
| 팀 성과 | mv_team_performance | org_id, 팀 |
| 포지셔닝 | mv_positioning_accuracy | org_id, positioning_strategy |
| 공고 모니터링 | recommended_bids | org_id, match_grade |
| RFP 캘린더 | calendar_items | org_id |

### 7.2 데이터 새로고침

- **페이지 로드**: 모든 위젯 데이터 조회
- **스코프 변경**: 선택 스코프의 데이터만 새로고침
- **위젯 on/off**: 해당 위젯만 새로고침
- **자동 새로고침**: 5분마다 (선택사항)

## 8. 성능 고려사항

### 8.1 최적화

- **병렬 조회**: 여러 위젯의 데이터를 병렬로 조회
- **페이지네이션**: 대량 데이터 위젯은 상위 N건만 표시
- **캐싱**: 스코프별 데이터 localStorage 캐싱
- **Lazy Loading**: 스크롤 시 하단 위젯 로드

### 8.2 로딩 상태

- 위젯 로드 중: 스켈레톤 또는 로딩 스피너
- 에러: 위젯별 에러 메시지 표시

## 9. 접근성 및 권한

### 9.1 권한 검증

**엔드포인트 호출 시 스코프 검증:**
- `/api/dashboard/team` → 사용자의 팀에만 접근 가능
- `/api/dashboard/division` → 사용자의 본부에만 접근 가능
