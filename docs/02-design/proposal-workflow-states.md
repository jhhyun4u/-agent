# 제안 프로젝트 상태 정의 및 워크플로우

## 📊 상태 정의

### 1️⃣ 대기 (WAITING)
- **진입 조건**: 신규 제안 프로젝트 생성
- **상태 유지 조건**:
  - PM/PL 담당자 할당 완료
  - 착수(진행) 버튼 미클릭
- **종료 조건**: 착수 버튼 클릭 → 진행 상태로 전환
- **특징**:
  - 제안팀, PM, PL 할당 필수
  - 입찰가 설정 필요
  - 마감일 기반 D-day 표시
- **UI 색상**: 🔘 회색 (#6B7280)

### 2️⃣ 진행 (IN_PROGRESS)
- **진입 조건**: 대기 상태에서 착수 버튼 클릭
- **상태 유지 조건**: 제안서 작성 중
- **실행 활동**:
  - RFP 분석
  - 제안서 기획 및 목차 수립
  - 섹션별 제안서 작성
  - 자가진단 및 검토
- **종료 조건**: 모든 섹션 작성 완료 → 완료 상태로 전환
- **특징**:
  - 다중 섹션 병렬/순차 작성
  - AI 지원 활성화
  - 섹션별 잠금/잠금 해제
  - 실시간 진행률 표시
- **UI 색상**: 🟢 초록 (#3ECF8E)

### 3️⃣ 완료 (COMPLETED)
- **진입 조건**: 진행 상태에서 모든 섹션 작성 완료 및 검토 통과
- **상태 유지 조건**: 최종 검토 단계
- **실행 활동**:
  - 전체 제안서 최종 검토
  - 컴플라이언스 체크
  - 자가진단 5개축 100점 평가
  - 문서 포맷 확인
- **종료 조건**:
  - 검토 통과 → 제출 상태로 전환
  - 재작성 필요 → 진행 상태로 역전환
- **특징**:
  - 읽기 전용 모드 (편집 불가)
  - 최종 다운로드 가능
  - 인쇄 포맷 확인
- **UI 색상**: 🔵 파랑 (#3B82F6)

### 4️⃣ 제출 (SUBMITTED)
- **진입 조건**: 완료 상태에서 제출 버튼 클릭
- **상태 유지 조건**: 발주기관 제출 후 평가 대기
- **실행 활동**:
  - 최종 HWPX/DOCX 변환
  - 제출 기관 업로드
  - 제출 기록 로그
  - 평가 현황 모니터링
- **종료 조건**:
  - 평가 대상 확인 → 발표 상태로 전환
  - 1차 탈락 → 종료 상태로 전환
- **특징**:
  - 제출 일시 자동 기록
  - 제출 현황 추적
  - 발주기관 평가일정 표시
- **UI 색상**: 🟡 주황 (#F59E0B)

### 5️⃣ 발표 (PRESENTATION)
- **진입 조건**: 제출 상태에서 평가대상 확정
- **상태 유지 조건**: 발표 평가 준비 중
- **실행 활동**:
  - 발표자료(PPT) 준비
  - 발표 스토리 및 메시지 개발
  - 리허설 및 Q&A 준비
  - 발표 팀 구성
- **종료 조건**:
  - 발표 완료 → 종료 상태로 전환
  - 발표 탈락 → 종료 상태로 전환
- **특징**:
  - PPT 리뷰 및 수정
  - 발표자료 버전 관리
  - 발표 일시 표시
  - 팀 멤버 역할 분담
- **UI 색상**: 🟣 자주 (#8B5CF6)

### 6️⃣ 종료 (CLOSED)
- **진입 조건**:
  - 발표 완료 후 평가 결과 등록
  - 또는 제출 탈락
  - 또는 발표 탈락
- **상태 유지 조건**: 평가 결과 및 피드백 정리
- **실행 활동**:
  - 평가 결과 등록
  - 수주/탈락 여부 기록
  - 제안 피드백 수집
  - 교훈 및 개선사항 정리
  - KB 업데이트 (수주 시 역량/콘텐츠, 탈락 시 경쟁사 정보)
- **종료 조건**:
  - 피드백 정리 완료 (자동 종료)
  - 또는 30일 자동 archive
- **특징**:
  - 읽기 전용 (편집 불가)
  - 성과 분석 데이터 표시
  - 교훈/피드백 입력
  - 성공/실패 원인 분석
- **UI 색상**: 🔴 빨강 (#EF4444)

### 🗂️ ARCHIVE (자동 이동)
- **진입 조건**: 종료 상태에서 30일 경과
- **자동 이동**: 시스템 스케줄러가 매일 자정에 자동 처리
- **특징**:
  - 목록에서 삭제 (아카이브 페이지에서 조회 가능)
  - 과거 데이터 통계에는 포함
  - 검색 결과에 포함 안 됨
  - 읽기 전용
- **UI**: 목록 필터에서 제외, 별도 "보관함" 섹션 제공

## ⏰ 자동 처리 및 모니터링

### 1. 마감일 모니터링
```javascript
- D-7: 경고 (주황색)
- D-3: 중요 (빨강색)
- D-0: 초긍박 (깜빡이는 빨강)
- D+1: 마감 경과 (회색 취소선)
```

### 2. 자동 Archive 처리
```javascript
// 매일 자정에 실행
- 종료 상태 → 30일 이상 경과
- status = 'archived'
- updated_at 기록
- 목록에서 제외
```

```javascript
- 각 상태별 AI 지원 활성화/비활성화

- 완료/제출/발표/종료: AI 기능 비활성화
```

## 📌 구현 노트

### Backend (app/graph/state.py)
```python
class ProposalStatus(str, Enum):
    WAITING = "waiting"           # 대기
    IN_PROGRESS = "in_progress"   # 진행
    COMPLETED = "completed"       # 완료
    SUBMITTED = "submitted"       # 제출
    PRESENTATION = "presentation" # 발표
    CLOSED = "closed"             # 종료
    ARCHIVED = "archived"         # 보관
```

### Database Migration
```sql
-- proposals 테이블 status enum 확장
ALTER TYPE proposal_status ADD VALUE 'submitted', 'presentation', 'closed', 'archived';

-- archived_at 컬럼 추가
ALTER TABLE proposals ADD COLUMN archived_at TIMESTAMP NULL;

-- 자동 archive 트리거
CREATE TRIGGER auto_archive_proposals
AFTER UPDATE ON proposals
FOR EACH ROW
WHEN (NEW.status = 'closed' AND NEW.closed_at < NOW() - INTERVAL '30 days')
EXECUTE FUNCTION archive_proposal();
```

## 🎯 핵심 개선사항

1. **명확한 프로세스**: 각 단계의 목적과 활동이 명확함
2. **자동화**: 30일 archive 자동 처리로 관리 비용 감소
3. **가시성**: 마감일 기반 색상 경고로 주의 필요 항목 즉시 파악
4. **감사추적**: 각 상태 전환 시점 기록으로 규정 준수
5. **지식축적**: 종료 후 교훈 입력으로 다음 제안에 활용
