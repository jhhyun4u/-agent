## 3. 재설계된 Knowledge Base 구조 (8 Categories)

### Category 1: 인력 자산 + 경력기술서 자동 생성
**"People Asset Database with CV Template Generation"**

#### 1-1 인력 DB (기존)
```
인력명 | 직급 | 전공 | 자격증 | 경력연수 | 주요경력 프로젝트 | 담당역할 | 언어능력
```

#### 1-2 경력기술서 서식 라이브러리 (신규)
```
공고별 요구 서식:
- 정부과제: 학력/경력/자격증 순서, A4 1-2매
- 민간과제: 자유로운 포맷
- 국방: 보안등급별 서식

자동 생성:
- 인력 DB → 공고 서식에 맞춰 경력기술서 자동 생성
- 예: RFP 요구 "AI 10년 경력" → 매칭 인력 자동 추천 + 경력기술서 자동 생성
```

#### 1-3 최신화 주기
- **분기별**: HR이 인력 정보 업데이트
- **서식**: 정부 규정 변경 시 즉시 업데이트

#### 1-4 활용 효과
- 시간 절감: 5-10시간/제안 (참여인력 정보 작성)

### Category 2: 수행실적 자산 + 실적증명서 자동 생성
**"Project Portfolio with Proof of Performance Templates"**

#### 2-1 수행실적 DB (기존)
```
프로젝트명 | 발주처 | 계약금액 | 기간 | 담당팀 | 주요성과 | 수행일자
```

#### 2-2 실적증명서 자동 생성 (신규)
```
공고별 요구 서식:
- 정부과제: 실적증명서 (양식 지정)
- 민간과제: 자유 포맷

자동 생성:
- RFP 분석 → 유사 프로젝트 자동 추천
- 추천 프로젝트 → 실적증명서 템플릿 자동 채움
- 예: "의료 AI 플랫폼" → 2023년 의료 프로젝트 자동 추천 → 증명서 자동 생성

추가 기능:
- 유사도 점수 (RFP와의 매칭도)
- 발주처별 추천 (같은 고객의 과거 프로젝트)
- 금액대별 추천 (예산 규모 비슷한 프로젝트)
```

#### 2-3 최신화 주기
- **프로젝트 완료 후 즉시**: PM이 프로젝트 정보 입력
- **분기말**: 검증 및 정리

#### 2-4 활용 효과
- 시간 절감: 3-5시간/제안 (실적증명 자료 준비)

### Category 3: 제출서류 템플릿 라이브러리
**"Submission Document Templates"**

#### 3-1 포함 내용
```
A. 법률/행정 서식
  ├─ 비밀유지서약서 (보안등급별: 일반/상대비밀/국방)
  ├─ 참여현황서 (인력/비용/기간)
  ├─ 자격요건 확인서 (기술능력, 재무능력)
  ├─ 기술안전관리계획
  └─ 기타 정부 규정 서식

B. 공고유형별 서식
  ├─ 정부과제 (나라장터 표준 서식)
  ├─ 국방과제 (보안 관련 추가 서식)
  ├─ 연구용역 (과제관리비, 성과관리 양식)
  └─ 민간과제 (자유 포맷)

C. 선택적 서식
  ├─ 재무건전성 선언
  ├─ 기술보안계획
  ├─ 환경친화성 계획
  └─ 사회적책임(CSR) 계획
```

#### 3-2 자동 생성 로직
```
RFP 업로드 → 공고유형 자동 분류 → 필요 서식 자동 추출
  → 해당 템플릿 + 가이드 제시
  → 가능한 항목 자동 채움 (회사정보, 인력정보 등)
```

#### 3-3 최신화 주기
- **반기별**: 정부 규정 변경 반영
- **즉시**: 새로운 규정 또는 공고유형 추가

#### 3-4 활용 효과
- 시간 절감: 2-3시간/제안 (각종 서식 작성)

### Category 4: Q&A 아카이브 + 발표 시사점
**"Q&A Archive with Presentation Insights"**

#### 4-1 포함 내용
```
A. 평가위원 질의 아카이브
  ├─ 기술 Q&A (기술 타당성, 리스크)
  ├─ 가격 Q&A (가격 정당성, 원가 기준)
  ├─ 일정 Q&A (일정 가능성, 마일스톤)
  ├─ 조직/인력 Q&A (전문성, 이행능력)
  └─ 발주처별 Q&A (특정 고객 특성)

B. 평가위원 질문 분석
  ├─ 질문 빈도 (자주 나오는 질문)
  ├─ 질문 패턴 (발주처별, 프로젝트 타입별)
  └─ 질문 난이도 (쉬운 질문 vs 함정 질문)

C. 발표 시사점 (신규)
  ├─ 이전 발표의 성공/실패 요인
  ├─ 평가위원의 반응 (질문 많음 = 관심 높음)
  ├─ 평가 점수와 Q&A의 상관관계
  └─ "이런 질문이 나오면 이렇게 답하니 점수 높음"
```

#### 4-2 데이터 구조
```
분류 | 발주처 | 프로젝트명 | 질문 | 우리답변 | 평가위원반응 | 낙찰여부 | 기술점수
  ↓
자동 분석 결과:
  - 질문 난이도 점수 (1-10)
  - 질문 관련도 (우리 제안과의 관련성)
  - 답변 품질 점수 (평가점수와의 상관관계)
  - "이 질문이 나오면 기술점수가 평균 X점"
```

#### 4-3 발표 시 활용
```
현장발표 예정 → KB 검색 "국방부 AI 이상탐지"
  ↓
결과 반환:
1. 유사 과제의 Q&A 5건
   - "보안은 어떻게 보증하나?" (난이도 7/10, 빈도 높음)
   - "일정 내 완성 가능한가?" (난이도 5/10)
   
2. 시사점
   - "이전 국방과제에서 보안 질문 많았음 → 기술점수 차이"
   - "우리 답변 'OOO' 방식이 점수 5점 높음"
   
3. 예상 Q&A 기본안
   - 질문 3개 제시 + 성공한 답변 템플릿
```

#### 4-4 최신화 주기
- **발표 직후**: 평가위원 질문 기록 (담당팀)
- **결과 공시 후**: 평가 점수와 함께 기록 (PM)

#### 4-5 활용 효과
- 시간 절감: 2-3시간/발표 (Q&A 준비)
- 발표 품질 향상: 예상 질문과 성공 사례에 기반한 준비

### Category 6: 가이드라인 & 평가기준 역설계
**"Guidelines & Evaluation Framework"**

#### 6-1 포함 내용 (기존)
- 제안서 작성 가이드
- 공공조달 규정
- 예타지침, 예산편성지침, 성과관리지침

#### 6-2 신규 추가: 평가기준 역설계
```
A. 공고유형별 평가배점 분석
  ├─ 정부R&D: 기술점수 60% vs 가격점수 40% (평균)
  ├─ 국방과제: 기술점수 70% vs 가격점수 30% (평균)
  ├─ 민간과제: 기술 40% vs 가격 60% (경향)
  └─ 변수: 발주처, 연도, 금액대

B. 각 평가항목별 점수 판정 기준
  ├─ 기술점수 80점 이상: 어떤 요소가 필요한가?
  ├─ 가격점수 90점 이상: 어떤 원가 기준을 사용하나?
  └─ "유사과제 분석 결과 X 요소가 점수 ±5점 영향"

C. 발주처별 특성 분석
  ├─ "국방부는 보안을 매우 중시 (기술점수 차이 큼)"
  ├─ "삼성 과제는 가격 민감 (가격점수로 당락 결정)"
  └─ "정부 과제는 예산 편성 기준 엄격함"
```

#### 6-3 최신화 주기
- **공고 공시 후**: 평가기준 변경 사항 반영
- **반기별**: 지난 공고 결과 분석 반영

### Category 7: 성과분석 대시보드 + 낙찰가 예측
**"Performance Analytics & Pricing Intelligence"**

#### 7-1 기본 성과분석 (기존)
- 낙찰률 (전사, 팀별, 분야별, 발주처별)
- 포지셔닝별 승률 (offensive vs defensive)
- 평가배점 추이

#### 7-2 신규 추가: 낙찰가 분석 및 가격 추천 (매우 중요)
```
A. 유사과제 낙찰가 DB
  프로젝트명 | 발주처 | 계약금액 | 낙찰가 | 낙찰률(%)
  의료 AI 플랫폼 | 삼성병원 | 50억 | 45억 | 90%
  국방 이상탐지 | 국방부 | 100억 | 95억 | 95%
  클라우드 전환 | 서울시 | 30억 | 28억 | 93%

B. 가격 범위 추천 로직
  RFP 입력 → 유사 과제 추천 → 낙찰률 기반 가격 범위 계산
  
  예시:
  "의료 AI, 50억 과제"
  → 유사 과제 5건 분석
  → 평균 낙찰률: 92% (범위: 85-95%)
  → 추천 입찰가: 46-48억 (확률 70%+)
  → "45억 아래로는 채산성 문제, 50억 그대로는 낙찰 확률 5% 미만"

C. 발주처별 가격 민감도
  ├─ 국방부: 가격 민감도 낮음 (기술이 먼저, 가격은 상한선)
  ├─ 민간기업: 가격 민감도 높음 (가격 경쟁 치열)
  ├─ 지방정부: 예산 편성 기준 엄격함 (원가 이상일 수 없음)
  └─ "이 발주처는 낙찰가 범위 X-Y 사이에 80% 집중"

D. 포지셔닝별 예상 낙찰률
  Offensive (기술 우위 강조):
  - 경쟁력 있는 분야: 70% 낙찰률
  - 경쟁력 약한 분야: 40% 낙찰률
  
  Defensive (가격 우위 강조):
  - 가격 우위 있음: 60% 낙찰률
  - 가격 경쟁력 약함: 30% 낙찰률
  
  "이 과제는 offensive로 가니 확률 70%, defensive로 가니 확률 35%"
```

#### 7-3 최신화 주기
- **낙찰 결과 확정 후 즉시**: 낙찰가 및 포지셔닝 기록
- **월별**: 가격 모델 재학습 (최근 추세 반영)

#### 7-4 활용 효과
- 시간 절감: 2-3시간/제안 (낙찰가 예측 및 포지셔닝)
- 낙찰 확률 향상: 데이터 기반 가격 책정

### Category 8: 제안 프로젝트 자산 (선택적 저장)
**"Proposal Project Assets (Reference Only)"**

#### 8-1 저장 대상 (재사용성 낮음, 참고용)
```
- 공고서/RFP: 참고 및 법적 증거용
- 리서치 자료: 유사분야 과제 시 기술 참고 (선택적)
- 작업 문서: 최종본만 보관 (초안은 3개월 후 폐기)
- 평가 결과: 통계 분석용 (비식별화)
```

#### 8-2 최신화 주기
- **참고 목적**: 별도 업데이트 필요 없음
- **폐기 정책**:

## 6. 추가 논의사항

### 6.1 기술 구현 이슈

**Q1. 경력기술서 자동 생성**
- 공고마다 서식이 다른데, 어디까지 자동화 가능한가?
- 정부: 표준 양식 O → 자동화 80%+
- 민간: 자유 포맷 → 자동화 50% (구조만 생성, 내용은 수정 필요)

**Q2. 실적증명서 자동 생성**
- 유사도 판정을 어떻게 하는가? (AI 분석 vs 키워드)
- 추천된 프로젝트가 신뢰도 있는가?

**Q3. 낙찰가 예측 모델**
- 유사과제 데이터가 충분한가? (최소 50개 이상 필요)
- 가격 모델 재학습 주기? (월별 vs 분기별)

---

### 6.2 데이터 수집 이슈

**Q1. Phase 0 (Cold Start)**

| Category | 필요 데이터 | 수집 방법 | 담당자 |
|----------|-----------|---------|--------|
| 1 | 50-70 인력의 경력정보 + 서식 사례 | HR + 수작업 | HR |
| 2 | 30-50 과거 프로젝트 + 낙찰가 정보 | PM + 회계 | PM/Finance |
| 3 | 정부/민간 제출 서식 샘플 | 수집/컴파일 | Compliance/Proposals |
| 4 | 50-100 과제의 Q&A + 평가점수 | 팀 노트 정리 | Proposals team |
| 6 | 평가배점 데이터 (100+ 공고) | 공고 분석 | PM |
| 7 | 50-100 낙찰가 기록 | Finance | Finance |

**예상 수집 기간**: 3-4주

**Q2. 데이터 정확성**
- 과거 프로젝트의 정보가 정확한가? (중복, 오류 가능성)
- 평가점수 데이터가 공시되지 않은 경우?
- 낙찰가 정보의 신뢰도?

---

### 6.3 운영 이슈

**Q1. 경력기술서 자동 생성 후 수정 필요**
- 완벽한 자동 생성은 어렵고, 항상 10-20% 수정 필요
- 그럼에도 기본틀 생성이 50% 시간 절감

**Q2. 서식 변경 관리**
- 정부 규정이 자주 바뀌는데, 빠르게 대응 가능한가?
- 담당자 (Compliance) 지정 필요

**Q3. 낙찰가 예측의 부정확성**
- 시장 변화, 경쟁사 전략에 따라 예측 오차 가능
- "이 모델의 신뢰도는 70%"라고 명시 필요

### 6.4 컨설팅사 KB의 차이

**TENOPA KB vs Consulting Firm KB**

| 항목 | 컨설팅사 | TENOPA |
|------|---------|--------|
| 목적 | 제안 품질 표준화 | 제출 서류/발표 준비 시간 단축 |
| 핵심 자산 | 성공 사례, 프레임워크 | 경력기술서, 실적증명서, Q&A |

**TENOPA의 강점**: 반복되는 서식 작성을 자동화 → 시간 절감 정량 측정 가능
