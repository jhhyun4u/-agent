## Executive Summary

| 항목 | 내용 |
|------|------|
| **문제정의** | 제안서 품질이 "인력 자산, 과거 실적, 회사 정보"의 **최신성과 정확성**에 크게 의존하나, 현재 이들 자료가 산재되어 있고 주기적 업데이트가 미흡함 |
| **솔루션** | 제안 프로세스의 각 단계별로 필요한 지식 자산(인력DB, 수행실적, 제안서 템플릿, 가이드라인 등)을 **한 곳에 통합 관리**하고, **자동 추천** 및 **주기적 현행화** 체계 구축 |
| **핵심가치** | 제안 작성 시간 30% 단축 + 제안서 품질 향상(신뢰성 강화) + 낙찰률 상향 |
| **범위** | 인력DB, 수행실적, 템플릿/가이드라인, 회사소개자료, Q&A, 승률대시보드 통합 관리 |
| **기대효과** | 1단계: 내부 협업 효율 향상 → 2단계: 제안 경쟁력 강화 → 3단계: 조직 학습 자동화 |

## 1. 현황분석 (As-Is)

### 1.1 제안서 작성에 필요한 정보 (실제 필요)

#### A. 인력 자산 정보
- **필요정보:** TENOPA 인력의 최신 경력기술서(이력서)
  - 학력, 자격증, 경력, 담당 역할, 전문분야
  - RFP의 "요구인력 사항"에 매칭
- **현황:** 개별 인력이 관리 중 → 최신성 보장 불가, 제안서마다 수집 필요
- **문제:** 인력정보 누락/중복, 작성 시간 소비, 품질 편차

#### B. 수행실적 자산
- **필요정보:** 최근 3~5년 유사 프로젝트 수행실적
  - 프로젝트명, 발주처, 금액, 기간, 담당자, 결과/성과
  - RFP의 "실적 증명" 요구사항 충족
- **현황:** 산재된 문서(엑셀, 이메일, 폴더) → 검색 어려움, 중복
- **문제:** 유사 프로젝트 찾기 어렵고, 증명 서류 작성에 시간 소비

#### C. 제안서 & 발표자료
- **필요정보:** 과거 잘 작성된 제안서, PPT, 스토리보드
  - 구조(목차), 섹션별 콘텐츠, 표현 방식, 디자인
  - "같은 유형의 과거 제안서"를 참고
- **현황:** 폴더 구조 복잡, 어느 것이 모범 사례인지 불명확
- **문제:** 제안서 작성 시간 오래 걸림, 스타일 편차 큼

#### D. 질의응답 아카이브
- **필요정보:** 유사 프로젝트의 "예상 질의 & 답변"
  - 기술 질문, 가격 정당성, 일정 가능성, 리스크 관리 등
  - 평가위원이 자주 물어보는 질문들
- **현황:** 개별 팀이 메모만 유지 → 조직 차원 공유 없음
- **문제:** 같은 질문에 다른 답변, 품질 편차

#### E. 회사소개 자료
- **필요정보:** 최신 회사소개자료, 조직도, 실력 부각 자료
  - 회사 역사, 비전, 핵심역량, 사업실적
  - "회사가 왜 유능한가"를 설득하는 자료
- **현황:** 마케팅/HR이 관리 → 제안 팀과 정보 불일치
- **문제:** 제안서 속 회사소개 섹션이 식상하고 일관성 없음

#### F. 가이드라인 & 정책 문서
- **필요정보:** 
  - 제안서 작성 가이드(형식, 문법, 단어 사용 규칙)
  - 참고 지침: 예비타당성조사지침, 성과관리지침, 예산편성지침
  - 평가배점 역설계 프레임워크
  - 공공조달 규정 (나라장터, 조달청 지침)
- **현황:** 산재된 외부 자료, 내부 가이드 미흡
- **문제:** 규정 위반으로 탈락, 가이드 불명확

#### G. 승률 분석 데이터
- **필요정보:** 
  - 입찰 결과 (낙찰/낙선/탈락)
  - 평가 배점 분석 (기술점수 vs 가격점수)
  - 포지셔닝별 승률 (offensive/defensive/adjacent)
  - 시간대별, 팀별 성과 추이
- **현황:** 산발적 기록 → 대시보드 없음
- **문제:** 실적 분석 어려움, 전략 개선 근거 부족

### 1.2 문제 종합

| 정보 유형 | 현황 | 문제점 | 영향 |
|---------|------|--------|------|
| 인력DB | 분산 관리 | 최신성 보장 불가 | 제안서 작성 지연 |
| 수행실적 | 산재된 문서 | 검색 어려움 | 실적 증명 시간 소비 |
| 제안서 템플릿 | 폴더 복잡 | 모범사례 불명확 | 작성 시간 증가 |
| Q&A | 개별 메모 | 조직 공유 없음 | 품질 편차 |
| 회사소개 | 부서별 관리 | 정보 불일치 | 신뢰성 약화 |
| 가이드라인 | 산발적 | 정책 통합 부족 | 규정 위반 리스크 |
| 성과분석 | 기록 미흡 | 대시보드 없음 | 전략 개선 어려움 |

**결과:** 제안 작성 프로세스가 **정보 수집 → 정리 → 작성**에서 30% 이상의 시간을 정보 검색에 소비

## 2. 목표 정의 (To-Be)

### 2.1 핵심 목표 (3단계)

**1단계: 정보 통합 (내부 협업 효율)**
- 산재된 제안 관련 정보를 **한 곳에 통합**
- 인력, 실적, 템플릿, 가이드를 **쉽게 검색**
- 현행화 주기 명확 (누가, 언제, 어떻게 업데이트하는가)

**2단계: 맥락 추천 (제안 품질 강화)**
- RFP 정보 기반 **자동 추천**
  - "이 프로젝트 유형에는 이 인력이 적합"
  - "유사 과제는 이거. 실적서 참고"
  - "같은 발주처 Q&A"
- 제안서 작성 시간 **30% 단축**

**3단계: 학습 자동화 (조직 경쟁력)**
- 수주/낙선 결과 기반 **자동 피드백**
  - "이 포지셔닝은 75% 낙찰률" (데이터 기반)
  - "이 발주처는 가격 민감" (평가이력 분석)
- 제안 전략 개선 **근거 제공**

### 2.2 SUCCESS CRITERIA

| # | 지표 | 측정 방법 | 목표 |
|---|------|---------|------|
| **SC-1** | 제안 작성 시간 | 제안 시작~제출 소요 시간 추적 | 30% 단축 (예: 40일 → 28일) |
| **SC-2** | 정보 검색 시간 | 제안팀 설문 (정보 수집 소요시간) | 기존 대비 60% 감소 |
| **SC-3** | 제안 품질 | 제안서 자가진단 점수 + 평가위원 피드백 | 평균 기술점수 3점 상향 |
| **SC-4** | 낙찰률 | 입찰 결과 분석 (낙찰/낙선) | 기존 대비 5%p 상향 |
| **SC-5** | 정보 최신성 | KB 업데이트 이력 추적 | 분기별 100% 업데이트 검증 |
| **SC-6** | 채택률 | 제안팀 사용률 조사 | 제안팀의 60% 이상 정기 사용 |

## 3. Knowledge Base 분류 체계 (What)

### 3.1 7가지 지식 카테고리

#### **Category 1: 인력 자산 (People Asset Database)**

**서브카테고리:**
- 1-1. 경력기술서 (이력서, 자격증, 보유역량)
- 1-2. 프로젝트별 담당자 이력 (누가 어떤 프로젝트 수행)
- 1-3. 역량 맵 (기술분야별 전문가)

**데이터 구조:**
```
인력명 | 직급 | 전공 | 자격증 | 경력연수 | 주요경력 프로젝트 | 담당역할 | 언어능력 | 최종업데이트일
```

**현행화 주기:** 분기별 (인사팀 + 본인 검증)  
**활용:** RFP의 "요구인력" 조건 매칭 → 자동 추천

---

#### **Category 2: 수행실적 자산 (Project Portfolio & References)**

**서브카테고리:**
- 2-1. 프로젝트 수행실적 (3~5년, 유사 프로젝트)
- 2-2. 발주처별 실적 (같은 고객의 과거 프로젝트)
- 2-3. 기술분야별 실적 (IoT, AI, 클라우드 등)

**데이터 구조:**
```
프로젝트명 | 발주처 | 계약금액 | 기간 | 담당팀 | 주요성과 | 첨부증빙 | 수행일자 | 비고
```

**현행화 주기:** 분기말 (프로젝트 종료 후 즉시 + 분기 검증)  
**활용:** RFP와 유사한 과거 프로젝트 추천 → 제안전략 수립, 실적 증명 자동 생성

**서브카테고리:**
- 3-1. 제안서 모범사례 (분야별, 유형별)
- 3-2. 섹션별 템플릿 (사업개요, 기술방안, 조직인력, 예산)

**현행화 주기:** 수주/낙선 후 즉시 (모범사례 추가 또는 제거)  
**활용:** 같은 유형 제안서 작성 시 "시작 템플릿" 제공 → 작성 시간 50% 단축

## 11. 예상 ROI (투자 대비 효과)

### 비용
- 시스템 개발: 4주 × 1명 개발자 = 20인일 (~1,000만원)
- 초기 데이터 수집: 2주 × 2명 = 8인일 (~400만원)
- 총 비용: ~1,400만원

### 효과 (연간)
- 제안작성 시간 30% 단축
  - 연평균 제안 건수: 10건
  - 연간 절감 시간: 10건 × 40일 × 30% = 120일
  - 인건비 절감: 120일 × 300만원/일 ≈ **3.6억원**

- 낙찰률 5%p 향상 (추정)
  - 현재 낙찰률: 30%
  - 향상 후: 35% (+5%p)
  - 평균 수주액: 50억원
  - 추가 수주 기대액: 50억원 × (5/30) ≈ **8.3억원**

- 제안 품질 향상
  - 기술점수 향상 → 평가위원 신뢰 ↑
  - 간접 효과: 영업활동 효율화 → **2억원 추정**

**총 연간 효과:** 약 **13.9억원**

**ROI:** 13.9억원 / 1.4백만원 ≈ **990배** (1년)

TENOPA의 제안 경쟁력은 **인력(경력기술서), 실적(수행증명), 콘텐츠(제안서 템플릿), 정보(가이드라인)**의 최신성에 달려있다. 현재 이들이 산재되어 있어 **제안 작성 시간 손실**과 **품질 편차**를 초래하고 있다.

### 솔루션
7가지 카테고리(인력, 실적, 제안서, Q&A, 회사소개, 가이드라인, 성과분석)를 **통합 관리**하는 Knowledge Base를 구축하면:
1. **제안 작성 시간 30% 단축** (정보 검색 자동화)
2. **제안 품질 향상** (모범사례 자동 추천)
3. **낙찰률 5%p 향상** (데이터 기반 전략 수립)

### 투자효율
- 투자: 1.4백만원 (2-3개월)
- 연간 효과: 13.9억원
- **ROI: 990배**

### 다음 단계
1. **경영진 승인** (ROI 검토)
2. **Phase 0 시작:** 초기 데이터 수집 (2주)
3. **사용자 인터뷰** (선택사항): 제안팀 5명과 심층면담으로 추가 요구사항 파악
